Анализ температурных данных и мониторинг текущей температуры

Streamlit‑приложение для анализа исторических температурных данных, выявления сезонных закономерностей и аномалий, а также мониторинга текущей температуры через OpenWeatherMap API.

Проект выполнен в рамках домашнего задания по анализу временных рядов и разработке интерактивных аналитических приложений.

⸻

Цели проекта

		•	Проанализировать исторические данные температуры для разных городов
		•	Выявить сезонные закономерности и температурные аномалии
		•	Построить сглаженные временные ряды и долгосрочные тренды
		•	Получить текущую температуру через OpenWeatherMap API
		•	Сравнить текущую температуру с исторической нормой для сезона
		•	Реализовать интерактивное веб‑приложение на Streamlit



⸻

Используемые данные

Файл temperature_data.csv содержит:

	•	city — название города
	•	timestamp — дата (шаг 1 день)
	•	temperature — среднесуточная температура (°C)
	•	season — сезон (winter, spring, summer, autumn)

Данные синтетические, сгенерированы на основе реалистичных сезонных средних температур.

⸻

 Анализ исторических данных

1. Скользящее среднее

		•	Используется окно 30 дней
		•	Позволяет сгладить краткосрочные колебания температуры

3. Сезонная статистика

Для каждого города и сезона рассчитываются:

		•	средняя температура
		•	стандартное отклонение

3. Поиск аномалий

Температура считается аномальной, если:
	
	T < mean − 2σ  или  T > mean + 2σ

Аномалии визуально выделяются на временном ряду.

⸻

Распараллеливание анализа

Анализ по городам может выполняться:

	•	последовательно — для небольших объёмов данных
	•	параллельно (joblib.Parallel) — при большом числе городов

Вывод:

	•	при 1–2 городах разница минимальна
	•	при 10+ городах ускорение ~1.5–2 раза
	•	параллелизм оправдан при масштабировании

⸻

Мониторинг текущей температуры

Используется OpenWeatherMap API.

Реализованы два подхода:

	•	Синхронный запрос (requests)
	•	Асинхронный запрос (aiohttp, asyncio)

Вывод:

	•	для одного города проще синхронный запрос
	•	для нескольких городов асинхронный способ быстрее и масштабируемее

Текущая температура сравнивается с исторической нормой для текущего сезона.

⸻

Streamlit‑приложение

В интерфейсе реализовано:

	•	загрузка CSV‑файла с историческими данными
	•	выбор города из выпадающего списка
	•	ввод API‑ключа OpenWeatherMap
	•	отображение:
	•	описательной статистики
	•	временного ряда температур
	•	аномалий
	•	сезонных профилей (mean ± std)
	•	текущей температуры и её оценки (норма / аномалия)

Все графики выполнены интерактивно с использованием Plotly.
